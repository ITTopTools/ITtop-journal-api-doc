# Аутентификация

Конечная точка аутентификации — вход для получения JWT access token, необходимого для всех остальных конечных точек.

## Конечные точки

### POST /auth/login

Вход с учётными данными для получения JWT токенов для доступа к API.

**Тело запроса:**

| Поле | Тип | Описание |
|------|-----|----------|
| `application_key` | string | Ключ приложения (фиксированное значение) |
| `id_city` | number \| null | ID города (необязательно) |
| `username` | string | Логин ученика |
| `password` | string | Пароль ученика |

**Ответ:** Объект с полями `access_token`, `refresh_token`, `expires_in_access`, `expires_in_refresh`, `user_type`, `city_data`, `user_role`.

**Процесс аутентификации:**

1. Вызвать `POST /auth/login` с учётными данными
2. Извлечь `access_token` из ответа
3. Использовать его как `Authorization: Bearer <access_token>` для всех последующих запросов

Эту конечную точку можно вызывать напрямую из Swagger UI — воркер проксирует запрос к реальному API. Если реальное API недоступно, воркер переключается на фиктивные данные.

Подробные инструкции по использованию кнопки Authorize в Swagger UI см. в разделе [Аутентификация](../authentication.md).
