# Аутентификация

Раздел аутентификации — вход для получения JWT токенов, необходимых для всех остальных эндпоинтов.

## Эндпоинты

| Method | Path | Краткое описание | Ссылка |
|--------|------|------------------|--------|
| POST | `/auth/login` | Отправить учётные данные для входа | [Подробнее](login.md) |

## Процесс аутентификации

1. Вызвать `POST /auth/login` с учётными данными
2. Извлечь `access_token` из ответа
3. Использовать его как `Authorization: Bearer <access_token>` для всех последующих запросов

Этот эндпоинт можно вызывать напрямую из Swagger UI — воркер проксирует запрос к реальному API. Если реальное API недоступно, воркер переключается на фиктивные данные.

## Зависимости

Все остальные разделы API зависят от действительного JWT токена, полученного из этого раздела. Подробные инструкции по использованию кнопки Authorize в Swagger UI см. в разделе [Аутентификация](../../authentication.md).
