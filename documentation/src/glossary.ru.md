# Глоссарий

Термины и понятия, используемые в документации.

<a id="worker"></a>
### Worker
[Cloudflare Worker](mock-server.md) — бессерверный прокси между Swagger UI и реальным API. Добавляет CORS-заголовки, подставляет `Origin`/`Referer` и отдаёт mock-данные при отсутствии токена.

<a id="spec"></a>
### Spec
Предмет (дисциплина) в учебном плане. В API встречается как `spec_id`, `spec_name`, `id_spec`, `name_spec`. Используется в эндпоинтах ДЗ, прогресса и расписания.

<a id="stream"></a>
### Stream
Учебный поток — набор студентов, обучающихся вместе. Поток содержит несколько [групп](#group). Идентифицируется `stream_id` и `stream_name`.

<a id="group"></a>
### Group
Учебная группа внутри [потока](#stream). Студенты одной группы делят расписание и ДЗ. Идентифицируется `group_id` или `current_group_id`.

<a id="dz"></a>
### DZ
Домашнее задание (дз). В UI — вкладка «ДЗ». В API `type=0` — обычное ДЗ, `type=1` — лабораторная работа.

<a id="anonymization"></a>
### Анонимизация
Процесс замены реальных персональных данных (имёна, ID, даты) на сгенерированные значения через [Faker](#faker). Гарантирует отсутствие чувствительных данных в публичном репозитории и mock-сервере.

<a id="faker"></a>
### Faker
Библиотека Python (`faker`) для генерации фиктивных данных. Используется на шаге `anonymize` пайплайна для подмены имён, сдвига дат и т.д.

<a id="pydantic"></a>
### Pydantic
Библиотека валидации данных Python. Модели в `src/validator/models.py` описывают ожидаемую структуру каждого ответа API. При изменении API валидация падает и открывает Issue в GitHub.

<a id="bearer-token"></a>
### Bearer-токен
Формат авторизации HTTP. После логина `access_token` отправляется как `Authorization: Bearer <token>` с каждым запросом. См. [Аутентификация](authentication.md).

<a id="swagger-ui"></a>
### Swagger UI
Интерактивный интерфейс документации API, встроенный в этот сайт. Работает на базе `openapi.json` и плагина `mkdocs-swagger-ui-tag`. См. [Swagger UI](swagger.md).

<a id="cors"></a>
### CORS
Cross-Origin Resource Sharing — механизм безопасности браузера. Журнал API не разрешает кросс-доменные запросы, поэтому [воркер](#worker) добавляет CORS-заголовки к ответам.

<a id="pipeline"></a>
### Пайплайн
Автоматизированный процесс сбора и публикации данных: `tests → collect → validate → anonymize → publish → worker → mkdocs`. Запускается ночью через GitHub Actions.
