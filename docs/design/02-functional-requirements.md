## Фичи по модулям

Разбей по логическим блокам приложения.

### [Модуль 1 — Сборщик (Collector)]

- [ ] Скрипт на playwright имитирует вход и сохраняет сетевой трафик (HAR) или просто делает GET/POST запросы к известным эндпоинтам.
- [ ] Выход: Набор JSON-файлов (сырые данные).

### [Модуль 2 — Валидатор и Анализатор (Pydantic Layer)]

- [ ] Действие: Проверка сырых данных на соответствие существующим моделям.
- [ ] Alert System: Если данные не прошли валидацию, GitHub Action падает и шлет уведомление(Issue). Это сигнал, что структуру пора обновлять.

### [Модуль 3 — Трансформер (Anonymizer)]

- [ ]     Действие: Самый важный этап для безопасности. Скрипт проходит по JSON и меняет реальные значения на синтетические (Faker).

        "FIO": "Иванов Иван" → "FIO": "James Smith"

        "ID": 12345 → "ID": 99999

- [ ] Выход: Очищенные JSON-примеры.
### [Модуль 4 — Генератор и Издатель (Publisher)]

- [ ] Действие: Скрипт (например, на базе FastAPI.openapi() или кастомный генератор) собирает openapi.json, вставляя в него очищенные примеры.

- [ ] Deployment: Обновленный файл пушится в GitHub Pages.

## Граничные случаи

Что происходит когда что-то идёт не так?

| Ситуация | Ожидаемое поведение |
|----------|-------------------|
| API Целевого ресурса было изменено | Валидация не пройдет успешно, openAPI не сможет создать документацию по причине того что схема отсутствует, алерт и генерация issie |
| API Целевого ресурса не отвечает | повторить попытку через 10 минут, 3 раза, в ином случае поптытатся проксировать трафик через общедоступные прокси сервера дабы избежать возможной блокировки, если ситуация не изменна, перестать опрашивать сервер на сутки, с пометкой в Openapi о том что сейчас Основной сервер недоступен |
| Данные не валидны для аутентификации | значит меня отчислили, можно не разрабатывать дальше |
| To many request | ретрай через 5 минут, при повторе - алерт и issue, поведение странное в связи с тем что сбор данных раз в сутки |
